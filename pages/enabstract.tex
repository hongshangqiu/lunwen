% English Abstract

With the widespread adoption of cloud computing and microservices, modern software systems have become larger and more complex. Traditional operations methods face many difficulties in fault diagnosis, including scattered knowledge, different types of data, and complex reasoning processes. Large language models have shown potential in this area, but they still have problems such as lacking domain-specific knowledge, missing real-time information, and giving unstable results.

To address these problems, this paper proposes a fault diagnosis method for software operations that combines retrieval-augmented generation, multi-source data fusion, and structured reasoning.

First, for the problem that operations knowledge is scattered and hard to use, this paper builds a knowledge base for operations scenarios and designs a document chunking strategy based on Markdown structure, along with metadata filtering for retrieval. Second, for the problem that logs, monitoring metrics and other data are in different formats and hard to use directly for reasoning, this paper designs a multi-source data fusion method that organizes retrieved results, log summaries, and abnormal metrics into a structured context. Third, for the problems of unstable results and poor explainability in root cause analysis, this paper designs a structured Prompt-based root cause analysis method that guides the large language model through the process of fault summary, evidence analysis, root cause inference, and repair suggestions.

To verify the effectiveness of the proposed method, this paper builds a prototype system and conducts experiments on 7 typical fault scenarios with 4 comparison groups. Experimental results show that using the large language model alone achieves about 35% accuracy. After adding knowledge retrieval, accuracy improves to 48%. With real-time logs and monitoring metrics added, accuracy reaches 63%. The complete method achieves 79% accuracy, with 93% Top-3 accuracy and 68% evidence coverage rate. These results show that combining retrieval augmentation, multi-source data fusion, and structured reasoning can effectively improve the accuracy and explainability of large language models in software operations fault diagnosis, providing a reference for using large language models to assist in operations analysis.
