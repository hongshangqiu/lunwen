% English Abstract

With the widespread adoption of cloud computing and microservice architectures, software operations fault diagnosis increasingly depends on heterogeneous runtime evidence. In practice, operators usually inspect monitoring dashboards and search abnormal logs at the same time. Logs provide event-level semantics, while monitoring metrics describe resource changes and performance trends. Relying on only one of them is often insufficient for stable root-cause localization. Directly applying large language models to operations diagnosis also suffers from limited evidence grounding, strong dependence on domain knowledge, and unstable outputs. Therefore, it is meaningful to study, under controlled conditions, whether collaborative analysis of logs and metrics can improve software operations fault diagnosis.

To address this problem, this thesis proposes a software operations fault diagnosis method based on collaborative analysis of logs and monitoring metrics. The method takes dual-source evidence as the main line. Logs are filtered and summarized with time windows and request correlation, while monitoring metrics are processed through anomaly detection and textual abstraction. The two sources are then organized into a unified analysis context. On top of this core pipeline, retrieval augmentation is introduced as an auxiliary module for knowledge-dependent scenarios, and structured reasoning is introduced as an output-control module to improve result consistency and auditability.

To evaluate the proposed method, a research validation prototype is implemented and an experimental framework is constructed with controlled scenarios as the main evidence and public samples as supplementary evidence. The main experiment contains 21 controlled samples from 7 fault families and compares five settings: Metric-Only, Log-Only, Log+Metric, Log+Metric+RAG, and Log+Metric+RAG+Structured. The Top-1 root-cause accuracy of the five settings is 38.1\%, 52.4\%, 76.2\%, 85.7\%, and 85.7\%, respectively. The results show that log-metric collaboration is the major source of improvement, retrieval augmentation further benefits middleware- and database-related scenarios, and structured reasoning mainly improves output compliance, raising it from 66.7\% to 95.2\% while also making key evidence more explicit. Supplementary validation on public log samples shows similar gains in output consistency.

The results indicate that, in the controlled setting of this thesis, a technical route centered on log-metric collaborative analysis, with retrieval augmentation and structured reasoning as auxiliary modules, can support software operations fault diagnosis more steadily. More complex fault propagation scenarios and larger-scale validation in real environments remain open issues for future work.
