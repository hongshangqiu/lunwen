% 英文摘要

{\small With the widespread adoption of cloud computing, microservices, and containerization, modern software systems have grown increasingly large and complex, posing significant challenges to traditional operations and maintenance workflows that rely heavily on manual expertise. To improve fault diagnosis efficiency and operational intelligence, this thesis designs and implements an LLM-based intelligent fault diagnosis prototype system for software operations --- LLM-Ops Copilot.}

{\small First, a domain-specific knowledge base for software operations is constructed. Based on the ChromaDB vector database, the knowledge base incorporates standard operating procedures, historical fault cases, and system architecture documentation, achieving vectorized storage, semantic retrieval, and metadata filtering by service name and fault type, thereby enhancing the model's ability to acquire task-relevant knowledge. Second, a multi-source operations data fusion method is proposed. The EvidenceBuilder module integrates retrieved knowledge, log summaries, and monitoring metric descriptions into a unified analysis context, including sub-steps such as anomaly metric extraction, severity assessment, and context construction. Third, an LLM-based Root Cause Analysis (RCA) engine is designed and implemented. Structured prompts guide the model to generate fault summaries, perform root cause inference, and provide repair suggestions, supporting confidence scoring, multi-hypothesis reasoning, and data gap identification, with a fallback rule-based analysis mechanism for robustness. Finally, comparative experiments are designed to evaluate the system effectiveness across multiple dimensions.}

{\small Experimental results demonstrate that relying solely on the LLM for direct responses leads to significant limitations in domain expertise, with an accuracy of only 43\%. Introducing RAG-based knowledge augmentation improves this to 57\%; further integrating real-time logs and monitoring metrics increases it to 71\%; and the complete method achieves 86\% accuracy with 100\% Top-3 accuracy and 78\% evidence coverage rate. Testing across seven typical fault scenarios including CPU overload, memory leak, database connection pool exhaustion, network latency, slow SQL query, service avalanche, and disk space insufficiency validates the feasibility of the proposed method in root cause localization accuracy, analysis completeness, and response efficiency, providing a novel technical pathway for intelligent operations and maintenance.}
