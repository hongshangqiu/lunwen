% English Abstract

With the widespread adoption of cloud computing and microservice architectures, software systems have become more distributed and operational diagnosis has become more dependent on heterogeneous evidence. In practice, operators usually inspect monitoring dashboards and search error logs at the same time. Logs provide event-level semantics, while monitoring metrics describe continuous runtime trends. Relying on only one of them is often insufficient for stable root-cause localization. Therefore, it is meaningful to study, under controlled conditions, whether collaborative analysis of logs and metrics can improve software operations fault diagnosis.

To address this problem, this thesis proposes a software operations fault diagnosis method based on collaborative analysis of logs and monitoring metrics. The method takes dual-source evidence as the main line. Logs are filtered and summarized with time windows and request traces, while monitoring metrics are processed through anomaly detection and textual abstraction. The two sources are then organized into a unified analysis context. On top of this core pipeline, retrieval augmentation is introduced as an auxiliary knowledge enhancement module for knowledge-dependent scenarios, and structured reasoning is introduced as an auxiliary output control module to improve evidence explicitness and output consistency.

To evaluate the proposed method, a research-oriented validation prototype is built and an experimental framework is designed with controlled scenarios as the main evidence and public samples as supplementary evidence. The main experiment contains 21 controlled samples from 7 fault families and compares five settings: Metric-Only, Log-Only, Log+Metric, Log+Metric+RAG, and Log+Metric+RAG+Structured. The Top-1 root-cause accuracy of the five settings is 38.1\%, 52.4\%, 76.2\%, 85.7\%, and 85.7\%, respectively. The Log+Metric setting performs more steadily than the two single-source settings, indicating that dual-source collaboration is the major source of improvement. Adding retrieval augmentation further improves knowledge-dependent scenarios such as database connection pool exhaustion and slow SQL diagnosis. Structured reasoning brings limited gains in accuracy, but raises output compliance from 66.7\% to 95.2\% and improves key-evidence hit rate from 81.0\% to 85.7\%. Supplementary validation on public Loghub samples further shows that structured output contributes to better consistency across datasets.

The results indicate that, in the controlled setting of this thesis, a technical route centered on log-metric collaborative analysis, with retrieval augmentation and structured reasoning as auxiliary modules, can effectively support software operations fault diagnosis. At the same time, long-chain fault propagation and larger-scale validation in real environments remain open issues for future work.
