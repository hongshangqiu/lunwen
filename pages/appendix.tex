\chapter{研究验证原型补充说明}

本文实现的研究验证原型代码托管于项目仓库。附录仅保留与论文方法和实验直接相关的结构化信息，用于说明原型如何支撑第三章的方法设计与第四章的实验评估。

\subsection{统一分析上下文示例}

原型中的分析上下文围绕“问题、时间窗、日志摘要、指标摘要、知识片段、严重程度”组织。其核心数据结构示例如下。

\begin{verbatim}
AnalysisContext(
    question="为什么该时间窗内接口延迟明显升高？",
    service="hotel-service",
    time_range="2026-03-11 09:00-09:15",
    metrics_summary="P95延迟持续上升，数据库连接数逼近上限",
    log_summary="出现 connection timeout 和 pool exhausted 报错",
    anomaly_indicators=["p95_latency", "db_connections"],
    retrieved_knowledge=[...],
    severity="high"
)
\end{verbatim}

\begin{verbatim}
{

    "suspected_root_causes": [
        {
            "cause": "数据库连接池耗尽",
            "confidence": 0.84,
            "supporting_evidence": [
                "连接数逼近上限",

            ],
            "contradicting_evidence": []

    "verification_steps": [

    ]

\end{verbatim}

\subsection{实验评分规则示例}

第四章采用统一评分规则对结果进行人工核对，其示例如下。

\begin{verbatim}
样例: 数据库连接池耗尽（轻度变体）
标准根因: 数据库连接池耗尽 / 连接泄露 / 连接池配置偏小
关键证据:
1. 数据库连接数接近上限
2. 日志出现 timeout 或 pool exhausted
3. 延迟明显上升

评分:
- Top-1命中: 是/否
- Top-3覆盖: 是/否
- 关键证据命中率: 命中条数 / 3
- 输出规范率: 必需字段完整数 / 4
\end{verbatim}
