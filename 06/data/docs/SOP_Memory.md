# 标准操作程序 (SOP): 内存泄漏与 OOM 故障处理 (ID: SOP-MEM-001)

## 1. 现象描述 (Symptoms)
- **指标异常**: 监控显示容器内存使用率持续攀升，且在垃圾回收 (GC) 后不回落。
- **系统日志**: 出现 `java.lang.OutOfMemoryError` 或 Python 的内存分配错误。
- **业务表现**: 接口响应时间变长，服务随机出现 502/504 错误。

## 2. 诊断步骤 (Diagnostics)
1. **查看堆栈**: 检查 `/logs/app.log` 中是否有 `Memory leak simulated` 或 `Heap dump created` 记录。
2. **进程检查**: 使用 `top -p [PID] -o %MEM` 确认内存占用最高的进程。
3. **频率分析**: 观察内存增长斜率，判断是突发性增长还是缓慢泄露。

## 3. 修复方案 (Mitigation)
- **紧急重启**: 重启受影响的服务节点。
- **模拟环境**: 若为故障模拟，请调用 `/fault/memory/leak` 清理接口。
- **代码修复**: 检查是否存在未关闭的文件句柄、数据库连接或全局静态集合。
